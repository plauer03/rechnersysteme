3. Wichtige Shell-Regeln & Variablen

Umleitungen

befehl > out.txt            # stdout überschreiben
befehl >> out.txt           # stdout anhängen (für Logs)
befehl 2> err.txt           # stderr (Fehler) umleiten
befehl > out.txt 2>&1       # stdout + stderr in eine Datei (Wichtig für Cron!)
befehl < input.txt          # Datei als Eingabe

Pipes und Verkettung

befehl1 | befehl2           # Ausgabe von 1 -> Eingabe von 2
befehl1 ; befehl2           # nacheinander
befehl1 && befehl2          # 2 nur wenn 1 erfolgreich (Exit-Code 0)
befehl1 || befehl2          # 2 nur wenn 1 fehlschlägt

Variablen

NAME="sensor"
echo "$NAME"
DATE=$(date +%Y%m%d_%H%M%S) # z.B. 20231025_143000
echo "$DATE"

Skript-Parameter

$0      # Skriptname
$1      # erster Parameter (z.B. ./skript.sh PARAM1)
$2      # zweiter Parameter
$?      # Rückgabewert des letzten Befehls (0 = OK, >0 = Fehler)

8. I2C in Python

Einzelne Register & Block lesen (z.B. BMP280)

#!/usr/bin/env python3
import smbus
import time

BMP280ADDR = 0x76
BUS = 1
bus = smbus.SMBus(BUS)

# Sensor konfigurieren
bus.write_byte_data(BMP280ADDR, 0xF4, 0x27)
bus.write_byte_data(BMP280ADDR, 0xF5, 0xA0)
time.sleep(0.5)

# 8 Byte Messdaten lesen ab Register 0xF7
data = bus.read_i2c_block_data(BMP280ADDR, 0xF7, 8)

# Rohdaten Bit-Shifting (Datenblatt-spezifisch)
adc_p = (data[0] << 12) | (data[1] << 4) | (data[2] >> 4)
adc_t = (data[3] << 12) | (data[4] << 4) | (data[5] >> 4)

9. Messwerte extrahieren und formatieren (Parsing)

Oft liefern Sensoren, Systembefehle oder andere Skripte Werte in einem Textformat, das erst bereinigt werden muss, bevor es in die RRD-Datenbank darf.

Variante A: Strings gezielt zerschneiden (.split() in Python)

Nutze diese Methode, wenn der Text immer exakt gleich aufgebaut ist. Fall: Du hast Ausgabe = "Leistung: 32000 W" und brauchst 32000 als Zahl.

ausgabe = "Leistung: 32000 W"

# 1. Am Doppelpunkt zerschneiden -> ["Leistung", " 32000 W"]
teile = ausgabe.split(":") 

# 2. Den zweiten Teil nehmen (Index 1)
wert_string = teile[1]     # " 32000 W"

# 3. Unnötige Zeichen entfernen (.replace und .strip für Leerzeichen)
wert_bereinigt = wert_string.replace("W", "").strip() # "32000"

# 4. In einen Zahlenwert (Integer oder Float) umwandeln
leistung = int(wert_bereinigt)

print(leistung) # Gibt aus: 32000

Variante B: Egal welcher Fall – Zahlen suchen (RegEx in Python)

Nutze diese Methode, wenn du dir unsicher bist, wo genau die Zahl steht. RegEx sucht gezielt nach Ziffern.

import re

# Verschiedene mögliche Antworten vom Sensor
zeile1 = "Temp=42.5'C"
zeile2 = "Leistung: 32000 W"
zeile3 = "Wert: -15.48 dbm"

# Die Zauberformel sucht nach: Optionales Minus, Ziffern, optionaler Punkt, Ziffern
muster = r"[-+]?\d*\.\d+|\d+"

# Für zeile1
match = re.search(muster, zeile1)
if match:
    temperatur = float(match.group())
    print(temperatur) # Gibt aus: 42.5

# Für zeile2
match2 = re.search(muster, zeile2)
if match2:
    leistung = int(match2.group())
    print(leistung) # Gibt aus: 32000

Variante C: Falsches Format normieren (z.B. DS18B20)

Fall: Der Sensor liefert 52123, gemeint sind aber 52,123 Grad Celsius. Hier muss der gelesene String in eine Zahl gewandelt und berechnet werden.

rohwert_str = "52123"

# 1. In Ganzzahl umwandeln
rohwert_int = int(rohwert_str)

# 2. Skalieren (durch 1000 teilen -> float)
temperatur = rohwert_int / 1000.0

# 3. Formatiert ausgeben oder direkt in RRD schreiben
print(f"Temperatur: {temperatur:.3f} °C") # Gibt aus: Temperatur: 52.123 °C

Variante D: Daten direkt in Bash auslesen (grep, awk)

Wenn du kein Python nutzt, sondern ein Bash-Skript schreibst.

ZEILE="Leistung: 32000 W"

# AWK: Nimmt das 2. Feld (getrennt durch Leerzeichen)
WERT_AWK=$(echo "$ZEILE" | awk '{print $2}')

# GREP: Zieht stur nur alle Ziffern aus dem String (-o = only matching, -E = extended regex)
WERT_GREP=$(echo "$ZEILE" | grep -o -E '[0-9]+')

echo "$WERT_AWK"   # Gibt aus: 32000
echo "$WERT_GREP"  # Gibt aus: 32000

10. RRDtool: Grundidee & Erstellung

RRD (Round Robin Database) reserviert beim Erstellen festen Speicherplatz. Damit GPRINT (MIN/MAX/LAST) im Graph exakt funktioniert, müssen die entsprechenden RRAs beim create definiert werden!

RRD erstellen (Korrektes Setup mit allen Archiven)

#!/bin/bash

rrdtool create /home/pi/temperatur.rrd --step 5 \
DS:temp1:GAUGE:600:-30:100 \
DS:temp2:GAUGE:600:-30:100 \
RRA:AVERAGE:0.5:1:17280 \
RRA:MIN:0.5:1:17280 \
RRA:MAX:0.5:1:17280 \
RRA:LAST:0.5:1:17280

Erklärung: DS:name:GAUGE:heartbeat:min:max (Datenquelle) RRA:FUNKTION:xff:steps:rows (Archiv) 1:17280 bedeutet: Jeder 1. Wert wird für 17280 Schritte (24h bei 5s) gespeichert.

11. RRD befüllen (Update)

Aus der Shell

rrdtool update /home/pi/temperatur.rrd N:23.40:24.10
# N = Now, danach folgen die Werte exakt in der Reihenfolge der DS.

Aus Python

import rrdtool

rrd_path = '/home/pi/temperatur.rrd'
t1 = 23.4
t2 = 24.1

# Format-String Methode
rrdtool.update(rrd_path, f'N:{t1:.2f}:{t2:.2f}')

12. RRD Graph erzeugen (Layering & GPRINT)

Regeln für saubere Graphen:

  1. Reihenfolge ist wichtig (Layering): RRDtool zeichnet von oben nach unten. Wenn du eine Fläche (AREA) und eine Linie (LINE) willst, musst du zuerst die AREA und danach die LINE schreiben. Sonst verdeckt die Fläche die Linie!
  2. Exakte GPRINTs: Lade für MIN, MAX und LAST jeweils einen eigenen DEF-Block, der exakt auf das korrekte RRA aus der Datenbank verweist.

Optimaler Graph mit AREA, LINE und allen GPRINTs

#!/bin/bash
filename="/home/raspi/databases/temperatur.rrd"
output_img="/var/www/html/cgi-bin/temperatur_graph.png"

rrdtool graph $output_img \
    --imgformat 'PNG' \
    --width 640 \
    --height 300 \
    --start now-10min \
    --end now \
    --vertical-label "Grad Celsius" \
    --alt-autoscale \
    --title "Temperaturverlauf DS18B20" \
    \
    DEF:t1_avg=$filename:temp1:AVERAGE \
    DEF:t1_min=$filename:temp1:MIN \
    DEF:t1_max=$filename:temp1:MAX \
    DEF:t1_last=$filename:temp1:LAST \
    \
    AREA:t1_avg#CCFFCC:"Bereich" \
    LINE2:t1_avg#0000FF:"Temperatur DS18B20\n" \
    \
    GPRINT:t1_last:LAST:"Aktuell\: %5.2lf °C" \
    GPRINT:t1_min:MIN:"Min\: %5.2lf °C" \
    GPRINT:t1_max:MAX:"Max\: %5.2lf °C" \
    GPRINT:t1_avg:AVERAGE:"Avg\: %5.2lf °C\n"

Achtung Syntax: Bei GPRINTs müssen Doppelpunkte, die als Text angezeigt werden sollen, escaped werden (\:). Das \n am Ende des LINE-Titels oder letzen GPRINTs sorgt für einen sauberen Zeilenumbruch in der Legende.

Graph für zwei Sensoren (Beide mit AREA und LINE)

Um beide sichtbar zu machen, sollte AREA eine leichte Deckkraft haben (Hex-Code mit 8 Zeichen: #RRGGBBAA, AA=Alpha/Transparenz, z.B. 55 für leicht durchsichtig). Alternativ zeichnet man einfach beide Flächen zuerst.

rrdtool graph test.png \
    --start now-1h --title "Zwei Sensoren" \
    DEF:t1=$filename:temp1:AVERAGE \
    DEF:t2=$filename:temp2:AVERAGE \
    AREA:t1#CCFFCC \
    AREA:t2#FFCCCC \
    LINE2:t1#00FF00:"Sensor 1" \
    LINE2:t2#FF0000:"Sensor 2"

13. Sensor -> RRD: Endlosschleife

Ein sauberes Python-Skript, das Sensoren ausliest, Daten parst und in die RRD schreibt.

#!/usr/bin/env python3

import time
import rrdtool
import re

RRD_PATH = '/home/pi/temperatur.rrd'

def lese_sensor_1():
    # Beispiel: Sensor gibt "Temp: 24.5 C" zurück
    raw_data = "Temp: 24.5 C"
    match = re.search(r"[-+]?\d*\.\d+|\d+", raw_data)
    return float(match.group()) if match else 0.0

while True:
    wert1 = lese_sensor_1()
    
    print(f"Update: {wert1:.2f} °C")
    rrdtool.update(RRD_PATH, f'N:{wert1:.2f}')
    
    time.sleep(5) # Muss zum --step der RRD passen!

15. CGI-Skript für RRD-Graph

CGI-Skripte schreiben das Bild direkt in den Web-Stream.

/var/www/html/cgi-bin/graph.cgi

#!/bin/bash

# WICHTIG: HTML/CGI-Header für Bilder!
echo "Content-Type: image/png"
echo ""

RRD="/home/pi/temperatur.rrd"

# Der Bindestrich '-' als Dateiname leitet das Bild nach stdout (ins Web)
rrdtool graph - \
    --imgformat PNG \
    --width 700 \
    --height 250 \
    --start -1h \
    --end now \
    --title "Live Temperatur" \
    DEF:t1=$RRD:temp1:AVERAGE \
    AREA:t1#CCFFCC \
    LINE2:t1#0000FF:"Temperatur"

Nicht vergessen: sudo chmod +x /var/www/html/cgi-bin/graph.cgi

16. Autostart mit cron

Befehle

crontab -e  # Editieren
crontab -l  # Anzeigen

Syntax & Beispiele

* * * * * befehl
│ │ │ │ │
│ │ │ │ └── Wochentag (0-6)
│ │ │ └──── Monat (1-12)
│ │ └────── Tag (1-31)
│ └──────── Stunde (0-23)
└────────── Minute (0-59)
# Alle 5 Minuten ausführen
*/5 * * * * /usr/bin/python3 /home/pi/script.py

# Beim Systemstart ausführen (Ausgabe in Log-Datei leiten!)
@reboot /usr/bin/python3 /home/pi/sensor.py >> /home/pi/cron.log 2>&1

Wichtige Regel: In cron immer absolute Pfade benutzen (/usr/bin/python3 statt python3).

17. Autostart mit tmux

tmux eignet sich hervorragend, um Skripte im Hintergrund laufen zu lassen, bei denen man später in der Konsole noch mitlesen möchte.

Basis-Befehle

tmux new -s sensor        # Neue Session namens "sensor" starten
tmux a -t sensor          # Wieder an "sensor" anhängen (attach)
tmux ls                   # Sessions anzeigen
tmux kill-session -t sens # Session beenden

Wichtige tmux Shortcuts (Innerhalb der Session)

Ctrl + b, d    # Detach (Session im Hintergrund weiterlaufen lassen)
Ctrl + b, c    # Neues Fenster in der Session
Ctrl + b, n    # Nächstes Fenster

Automatisiertes Start-Skript für tmux

#!/bin/bash
# Startet eine Session detached (-d) und führt das Python-Skript aus
tmux new-session -s "logger" -d "/usr/bin/python3 /home/pi/sensor.py"

18. Typische Mini-Skripte & Logging

Fehler und Ausgaben in Datei mitschreiben

Willst du im Terminal die Werte sehen und gleichzeitig in eine Datei loggen? Nutze tee.

python3 sensor.py | tee -a sensor.log

Bash-Wrapper für stabiles Python-Skript

Falls das Skript mal abstürzt, startet diese Schleife es sofort neu.

#!/bin/bash
while true; do
    echo "[$(date)] Skript startet..." >> fehler.log
    /usr/bin/python3 /home/pi/sensor.py >> fehler.log 2>&1
    sleep 5
done

19. Fehlerbilder, Diagnose & Prüfungsstrategie

Diagnose-Befehle

systemctl status lighttpd   # Läuft der Webserver?
tail -n 50 /var/log/lighttpd/error.log  # Webserver Fehler?
ps aux | grep python        # Läuft das Python-Skript im Hintergrund?
ls -l /dev/i2c* # Sind die I2C-Geräte verfügbar?

Typische Prüfungsfehler

  1. RRD bleibt leer oder Graph ist defekt: - Stimmt die Anzahl der Werte beim Update exakt mit den DS beim Create überein?
    • Sind die Werte innerhalb des beim Create definierten Bereichs (min:max)?
    • chmod +x beim CGI-Skript vergessen?
  2. CGI zeigt nur kaputtes Bild-Icon:
    • Hast du echo "Content-Type: image/png" und danach unbedingt echo "" im Skript stehen?
    • Hast du absolute Pfade zur .rrd-Datei im CGI-Skript verwendet? (Ein CGI hat /var/www/html als Startpunkt, relative Pfade wie ./daten.rrd schlagen fehl).
  3. Sensor-Werte falsch gelesen:
    • I2C Bus Adresse geprüft? (i2cdetect -y 1)
    • Beim Daten-Parsing (RegEx/Split) falschen Index genommen?

4-Schritte-Prüfungsstrategie

  1. Sensor isoliert testen: Nur das Python-Skript schreiben und Werte per print() im Terminal kontrollieren. Passt das Parsing?
  2. Datenbank anlegen & manuell testen: RRD erstellen und ein Update per Hand (rrdtool update ... N:20) testen.
  3. Verknüpfen: Das Python-Skript mit rrdtool.update() erweitern und laufen lassen.
  4. Visualisieren: Das Bash-Graph-Skript schreiben. Zuerst als lokales Bild (test.png) speichern und ansehen, erst wenn das klappt, ins /cgi-bin/ als Web-Ausgabe verschieben.
